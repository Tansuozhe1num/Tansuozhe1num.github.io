常用 GDB 命令

查看源码和设置断点

  • list (l)
    显示源码，默认显示当前停留位置附近的代码。
  • break (b)
    在指定的函数或行设置断点。
    示例：在 main 函数入口设置断点：

或在具体文件的某一行设置断点：

启动和控制程序运行

  • run (r)
    启动程序执行，可以附带命令行参数。
  • continue (c)
    当程序在断点处停止后，继续执行到下一个断点或程序结束。

  • step (s)
    单步执行代码，遇到函数调用时会进入该函数内部。

  • next (n)
    单步执行代码，但不会进入函数内部，而是整体执行完当前行的函数调用。

  • finish
    执行完当前函数，返回到调用该函数的地方，并暂停。

    查看变量和表达式

  • print (p)
    打印变量或表达式的值。

  • display
    设置表达式，使得每次程序暂停时自动显示其值。
  • info locals
    查看当前函数内所有局部变量的值。

查看调用堆栈

  • backtrace (bt)
    显示当前的调用堆栈，有助于追踪程序是如何进入当前状态的。

观察点和条件断点

  • watch
    设置观察点，当某个变量或内存位置的值发生变化时暂停程序。

  • rwatch 和 awatch
    分别用于在读取（rwatch）和读写（awatch）变量时触发暂停。

  • 条件断点
    仅在满足特定条件时才触发断点。
    例如，在 my_function.c 文件的第 50 行，当变量 x 等于 10 时才停下：

辅助命令

  • info breakpoints
    查看当前所有设置的断点信息。
  • delete
    删除指定编号的断点。
  • set args
    设置程序启动时的命令行参数（在启动 run 前设置）。
  • help
    获取某个命令的帮助信息。
  • quit (q)
    退出 GDB。
